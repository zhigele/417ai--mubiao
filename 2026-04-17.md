# 2026-04-17

## YOLOv8 检测系统 - 页面"正在连接服务器"不消失问题修复

### 问题根因
`MAIN_TEMPLATE` 的 `<script>` 块被一个大 `try { ... } catch(e){}` 包裹，导致：
1. `try` 内任意代码抛出异常 → 直接跳到 catch
2. `document.getElementById('modelSection').classList.remove('hidden')` 不被执行
3. `modelSection` 始终 hidden，页面永远转圈

### 修复
删除了 script 外层的 `try/catch` 包裹，各函数内部分别处理异常。

### 首页路由演变记录
- 最初：`render_template_string(MAIN_TEMPLATE)` ← 正常
- 中间误改：返回磁盘旧 `index.html` ← 导致前端代码版本不匹配
- 最终：直接字符串替换 `{{username}}` + 返回 HTML response，绕过 Jinja2

### 检测结果显示
- 后端 `/api/detect` 200 成功，返回 base64 图片
- 前端双栏布局：左侧标注图片 + 右侧类别统计/详细列表
- 图片加载用独立 id 控制 hidden/visible，不依赖 DOM 树结构

## 2026-04-17 再次修复：modelSection 仍卡在"正在连接服务器"

### 问题现象
- Flask 日志显示 `GET / 200` 成功
- 浏览器显示"正在连接服务器..."转圈
- `/api/health` 请求根本没出现在 Flask 日志中（fetch 被浏览器缓存/挂起）

### 根因
即便删了 try/catch 包裹，`modelSection` 的显示逻辑仍依赖 DOMContentLoaded + readyState 判断，健康检查 fetch 有时在某些浏览器缓存状态下不触发 `.then()`。

### 最终修复方案
用立即执行函数 (IIFE) 在 `<script>` 最顶部**同步**把 serverStatus 改为"服务就绪"并移除 `modelSection` 的 hidden，完全不等任何网络请求。健康检查仅用于更新状态栏附加信息（PyTorch版本、CUDA），加了 AbortController 4秒超时。

## 2026-04-17 第三次修复：IIFE 移至内联 script + Tailwind 换源

### 问题根因（最终确认）
国内访问 `cdn.tailwindcss.com` 超时，导致整个 HTML 的 `<script>` 块加载失败，IIFE 完全没机会执行。

### 修复措施
1. 把显示 modelSection 的 IIFE 从大 `<script>` 移入紧挨 `#modelSection` 的**内联 `<script>` 标签**（写在 `<div id="modelSection">` 之后，HTML 解析到这里就立即执行，不依赖任何外部 CDN）
2. IIFE 里用纯内联 style 替代 Tailwind 类，避免 Tailwind 未加载时样式失效
3. 把 Tailwind CDN 从 `cdn.tailwindcss.com` 换为 `cdn.jsdelivr.net/npm/tailwindcss-cdn@3.4.16`（国内可用）
